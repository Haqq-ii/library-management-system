## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, filter bar gaps |
| md | 16px | Default element spacing, card padding |

## Color

All values are OKLCH CSS variables from globals.css. Do not hardcode hex values — use Tailwind semantic tokens.

| Role | CSS Variable | OKLCH (light) | Usage |
|------|-------------|---------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) — white | Page background, table background |
| Secondary (30%) | `--card` / `--sidebar` | oklch(1 0 0) / oklch(0.985 0 0) | Report cards, stat cards, filter bar containers |
| Accent (10%) | `--primary` | oklch(0.205 0 0) — near-black | Column header sort chevrons (active), primary CTA buttons |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) — red | Overdue row highlight background (bg-red-50 / text-red-600), overdue badge |
| Muted | `--muted-foreground` | oklch(0.556 0 0) — medium gray | Captions, secondary text, empty state body, filter label text |

Accent reserved for:
- The "Run Report" / "Apply" primary action button (variant="default")
- Active sort column indicator (column header chevron)
- Active nav item indicator (border-l-2 border-slate-900 — existing AppSidebar pattern, unchanged)

Semantic color rules for this phase:
- Overdue loan rows: `bg-red-50` row background, `text-red-600 font-medium` for days-late value — matches LoansTable.tsx established pattern.
- Chart colors: use existing `--chart-1` through `--chart-5` CSS variables. Do NOT introduce new chart color tokens.
  - `--chart-1` (oklch(0.87 0 0), light gray): "Loans Issued" series
  - `--chart-2` (oklch(0.556 0 0), medium gray): "Loans Returned" series
  - Chart fill/stroke must reference `var(--chart-1)` and `var(--chart-2)` via the Tailwind `--color-chart-*` mappings.
- Fine summary badges:
  - Total recorded: no badge, plain `text-3xl font-bold`
  - Total waived: `bg-orange-100 text-orange-800` (matches FINE_WAIVED badge in AuditTable.tsx)
  - Total outstanding: `bg-red-100 text-red-800`

## Page Layout Contract

### /reports — Main Reports Page

```
<page>
  <h1 class="text-2xl font-semibold">Reports</h1>
  <Tabs defaultValue="overdue">
    <TabsList>
      <TabsTrigger value="overdue">Overdue Loans</TabsTrigger>
      <TabsTrigger value="popular">Popular Books</TabsTrigger>
      <TabsTrigger value="activity">Borrowing Activity</TabsTrigger>
      <TabsTrigger value="fines">Fine Summary</TabsTrigger>
    </TabsList>
    <TabsContent value="overdue">  ← RPT-01 </TabsContent>
    <TabsContent value="popular">  ← RPT-02 </TabsContent>
    <TabsContent value="activity"> ← RPT-03 </TabsContent>
    <TabsContent value="fines">    ← RPT-04 </TabsContent>
  </Tabs>
</page>
```

- Page wrapper: `space-y-6` (24px gap between page title and tabs) — consistent with all existing pages.
- Tab content panels: `space-y-4` between filter bar and table/chart.
- Grid for stat cards (RPT-04): `grid gap-4 md:grid-cols-3` — matches DashboardStats.tsx.

### RPT-01: Overdue Loans Tab

```
[Filter: sort by "Days Late" (desc default) | sort by "Member Name" | sort by "Title"]
[Table: Member | Book Title | Due Date | Days Late | Fine Accruing]
[Pagination: Prev — Page N of M — Next]
```

- Default sort: days late descending (most overdue first).
- Sort state: client-side `useState` — no server round-trip needed if all overdue loans fit in one load (expected <200 rows).
- Row highlight: `bg-red-50` on every row (all rows are overdue by definition — but still apply for consistency).
- "Days Late" cell: `text-red-600 font-medium`.
- Column header sort triggers: `<button>` with `aria-sort` attribute. Use `ChevronUp` / `ChevronDown` lucide icons (16px) alongside header label.

### RPT-02: Popular Books Tab

```
[Filter bar: From [date input] To [date input] — Apply button]
[Table: Rank | Title | Author | Borrow Count]
[Pagination: Prev — Page N of M — Next]
```

- Default date range: last 30 days (computed from today on server render).
- Rank column: plain number, `text-sm text-muted-foreground`, width `w-12`.
- Apply button triggers server refetch via `useTransition` — show `isPending` spinner on button.
- No empty-rank row — list starts at 1, no gaps.

### RPT-03: Borrowing Activity Tab

```
[Filter bar: From [date input] To [date input] — Apply button]
[Card: <AreaChart or LineChart from recharts, responsive, 320px height>]
[Legend: "Loans Issued" (chart-1) | "Loans Returned" (chart-2)]
```

- Chart component: `ResponsiveContainer width="100%" height={320}` wrapping a `LineChart`.
- Two data series: `loanCount` (loans issued) and `returnCount` (loans returned), grouped by day.
- X-axis: date label, `tick={{ fontSize: 12 }}`, show every 7th label if range > 14 days.
- Y-axis: integer count, `tick={{ fontSize: 12 }}`, `allowDecimals={false}`.
- Tooltip: default Recharts `<Tooltip />` with `contentStyle={{ fontSize: '12px' }}`.
- Chart wrapper: inside a `<Card>` with `<CardHeader><CardTitle>Borrowing Activity</CardTitle></CardHeader>`.
- Use `"use client"` directive on the chart component — Recharts requires client rendering.
- Default date range: last 30 days.

### RPT-04: Fine Summary Tab

```
[Grid: 3 stat cards]
  Card 1: "Total Fines Recorded"  — value in text-3xl font-bold
  Card 2: "Total Waived"          — value in text-3xl font-bold, badge bg-orange-100
  Card 3: "Total Outstanding"     — value in text-3xl font-bold, badge bg-red-100
[Divider]
[Summary note: "Outstanding = Recorded minus Waived" — text-sm text-muted-foreground]
```

- Currency display: `$0.00` format (two decimal places). Use `Number.toFixed(2)` — Prisma Decimal serialized to `number` in Server Action (matches FinesTable pattern).
- No date range filter on RPT-04 — shows all-time totals.

## Interaction States

### Loading State

- Table panels (RPT-01, RPT-02): show `<Skeleton className="h-8 w-full" />` repeated 5 times inside the table body area while `isPending`.
- Chart panel (RPT-03): show `<Skeleton className="h-[320px] w-full rounded-md" />` while `isPending`.
- Stat cards (RPT-04): show `<Skeleton className="h-10 w-24" />` inside each `CardContent`.

### Empty State

| Report | Empty State Heading | Empty State Body |
|--------|--------------------|--------------------|
| RPT-01 Overdue | "No overdue loans" | "All loans are currently on time." |
| RPT-02 Popular | "No borrowing data" | "No loans were issued in the selected date range. Try a wider range." |
| RPT-03 Activity | "No activity data" | "No loans or returns recorded in the selected date range. Try a wider range." |
| RPT-04 Fine Summary | (no empty state — zeros are valid data) | Show $0.00 in all three stat cards |

Empty state layout: `py-12 text-center` — heading `text-base font-semibold`, body `text-sm text-muted-foreground mt-1`. Matches AuditTable.tsx empty state pattern exactly.

### Error State

- Display pattern: replace table/chart area with a `text-center py-12` block.
- Heading: "Could not load report data"
- Body: "An error occurred while fetching this report. Refresh the page to try again."
- No destructive actions in this phase — no confirmation dialogs required.

## Accessibility Contract

- All sortable column headers: `<button>` element with `aria-sort="ascending"` / `aria-sort="descending"` / `aria-sort="none"`. Chevron icons are `aria-hidden="true"`.
- Chart (`<ResponsiveContainer>`): wrap in `<figure>` with `<figcaption className="sr-only">Borrowing activity chart showing loans issued and returned over time</figcaption>`.
- Date inputs: use `<label>` pairing (matching AuditTable pattern — `<label className="text-xs text-muted-foreground font-medium">`).
- Stat cards: each `<CardTitle>` is the accessible label for the value it describes — no additional aria needed.
- Tab panel: Radix Tabs handles `role="tabpanel"` and `aria-labelledby` automatically.
- Min touch target for sort buttons: `min-h-[44px]` (matches nav link pattern).
- Color is never the sole indicator of status — "Days Late" value uses both `text-red-600` color AND numeric text to convey overdue severity.

## Implementation Notes for Executor

1. **recharts install first**: `npm install recharts` must be the first task in the plan. All subsequent RPT-03 work depends on it.
2. **Server Actions pattern**: Follow the `getNotificationLog` / `getAuditLog` pattern — Server Actions return `{ success: true, data: ... }` or `{ success: false, error: string }`.
3. **Client component boundary**: Only the chart component and the sort-state table components need `"use client"`. The `/reports` page itself can be a Server Component that passes initial data as props (matching `NotificationsPage` / `AuditPage` pattern).
4. **Date serialization**: All dates passed from Server Actions to client components must be serialized to strings (ISO or locale string) — not `Date` objects. Match the existing `serializedFines` pattern in FinesPage.
5. **Prisma Decimal**: Fine amounts from Prisma are `Decimal` type — convert to `Number` in Server Action before returning (matches FinesPage `amount: Number(f.amount)` pattern).
6. **No new CSS tokens**: All chart colors use existing `--chart-1` / `--chart-2` CSS variables. No additions to globals.css.
7. **Page route**: `/app/(app)/reports/page.tsx` — librarian auth guard identical to other librarian pages.
