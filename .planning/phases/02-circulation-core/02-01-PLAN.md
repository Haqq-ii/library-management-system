<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Server Action (checkout) | Librarian-supplied memberId/bookId cross into a mutating, money/inventory-relevant transaction |
| browser → Server Action (search) | Free-text query reaches Prisma; must not leak member PII to non-librarians |
| concurrent requests → BookCopy row | Two checkouts may target the last AVAILABLE copy simultaneously |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | checkoutBook / searchMembers / searchBooks | mitigate | `requireRole("LIBRARIAN")` is the first call in every action (CVE-2025-29927 defense — middleware is not trusted) |
| T-02-02 | Tampering | checkoutBook input | mitigate | Zod `CheckoutSchema.safeParse` validates memberId/bookId before any DB write; INVALID_INPUT on failure |
| T-02-03 | Tampering (race) | BookCopy availability | mitigate | `SELECT ... FOR UPDATE SKIP LOCKED` inside `prisma.$transaction` guarantees exactly one checkout of the last copy |
| T-02-04 | Information Disclosure | searchMembers | mitigate | Search gated by `requireRole("LIBRARIAN")`; members cannot enumerate other members |
| T-02-05 | Denial of Service | search actions | accept | `take: 10` caps result size; no rate limit added in v1 (single-institution, authenticated librarian only — low risk) |
| T-02-SC | Tampering | shadcn CLI add (tabs/popover/command) | mitigate | shadcn copies vetted source into repo; no new runtime npm package. Radix deps are transitive and already present via existing shadcn components. No [ASSUMED]/[SUS] packages introduced |
</threat_model>
