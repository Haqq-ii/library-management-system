<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → returnBook Server Action | Librarian-supplied loanId triggers loan closure, fine creation, and reservation advancement |
| concurrent returns/checkouts → BookCopy + Reservation | Return mutates copy status and a reservation row that other transactions may touch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Elevation of Privilege | returnBook | mitigate | `requireRole("LIBRARIAN")` first line; middleware not trusted (CVE-2025-29927) |
| T-02-07 | Tampering | returnBook idempotency | mitigate | Guard rejects loans where `returnedAt` is already set (ALREADY_RETURNED) — prevents double-fine / double-return |
| T-02-08 | Tampering (race) | copy status + reservation advance | mitigate | All reads/writes (loan, fine, copy, reservation) occur inside a single `prisma.$transaction` so the hold advance is atomic with loan closure |
| T-02-09 | Repudiation | fine creation | accept | Full audit log of librarian actions is Phase 3 (AUD-01); fine row carries createdAt/memberId as a partial trail in v1 |
| T-02-10 | Tampering | fine amount | mitigate | Amount derived server-side from LoanPolicy.fineDailyRate × server-computed overdue days; client cannot supply the amount |
</threat_model>
