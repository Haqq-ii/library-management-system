## Spacing Scale

8-point scale. All values are multiples of 4. No new exceptions for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding (badge `px-2 py-0.5`) |
| sm | 8px | Compact element spacing, row action button gaps |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section padding, dialog content padding |
| xl | 32px | Layout gaps between page sections (`space-y-8`) |
| 2xl | 48px | Major section breaks, empty state `py-12` |
| 3xl | 64px | Page-level top padding (not used in this phase) |

Exceptions: Touch targets for nav links use `min-h-[44px]` (established in AppSidebar.tsx — accessibility minimum). Dialog action buttons use `gap-2` (8px) in `DialogFooter`. Pagination controls use `size="sm"` buttons.

### Fine Waiver Dialog

**Trigger:** Waive button on UNPAID row opens `<Dialog>`.

**Dialog structure:**
```
DialogTitle:       "Waive Fine"
DialogDescription: "Waive fine of $[amount] for [Member Name]? This action cannot be undone."
Body:              <Label>Reason (required)</Label>
                   <Textarea placeholder="Enter reason for waiving this fine..." required minLength={1} />
DialogFooter:      <Button variant="outline">Cancel</Button>
                   <Button variant="destructive" disabled={isPending || reason.trim().length === 0}>Confirm Waive</Button>
```

**Validation:** Confirm Waive button is disabled until reason textarea contains at least 1 non-whitespace character. No form library needed — controlled state with `useState`.

**On success toast:** `toast.success("Fine waived successfully.")`

**On error toast:** `toast.error("Couldn't waive fine. Please try again.")`

## Interaction States

### Loading States
- All new tables use `<Skeleton>` rows (3 skeleton rows at 40px height) while data loads. Match pattern in existing pages.
- Action buttons (`Waive`, `Renew`, `Cancel`, `Yes, Cancel`, `Confirm Waive`) use `disabled={isPending}` during Server Action execution.

### Disabled States
- `Waive` button: hidden (or `—`) on PAID and WAIVED fine rows — not disabled-visible, just absent.
- `Cancel` button: absent on READY, FULFILLED, CANCELLED reservation rows.
- `Renew` button: present on all active loan rows; blocking is enforced server-side with a toast error (not disabled client-side).
- `Reserve` button: disabled (`disabled={book.availableCount > 0 || isPending}`) when copies are available — existing behavior.

### Error States
- Server Action failures: Sonner `toast.error(...)` — existing pattern.
- FINE_BLOCK on checkout: Inline error paragraph inside `CheckoutSheet` below the member search area: `<p className="text-sm text-destructive mt-2">[message]</p>`.
- Dialog form validation: Confirm Waive button disabled until reason is non-empty — no visible error message needed (button state communicates the constraint).

### Optimistic Updates
- None in Phase 3. All mutations wait for Server Action result before updating UI. Use `useTransition` with `isPending` to show loading state.

## AuditLog `details` JSON Schema

The `details` Json field stores a human-readable `description` plus structured fields for each action type. The Description column in /audit renders `details.description` directly.

| AuditAction | `details` structure |
|-------------|---------------------|
| CHECKOUT | `{ description: "Checked out '[bookTitle]' (copy [barcode]) to [memberName], due [dueAt]", memberId, memberName, bookTitle, copyBarcode, dueAt }` |
| RETURN | `{ description: "Returned '[bookTitle]' (copy [barcode]) from [memberName]", memberId, memberName, bookTitle, copyBarcode, returnedAt }` |
| FINE_WAIVED | `{ description: "Waived $[amount] fine on '[bookTitle]' for [memberName]. Reason: [reason]", fineId, memberId, memberName, bookTitle, amount, reason }` |
| BOOK_ADDED | `{ description: "Added book '[title]' by [author] (ISBN: [isbn])", bookId, title, author, isbn }` |
| BOOK_EDITED | `{ description: "Edited book '[title]'", bookId, title }` |
| BOOK_DELETED | `{ description: "Deleted book '[title]'", bookId, title }` |
| MEMBER_ADDED | `{ description: "Registered member [name] ([email]) as [memberType]", memberId, name, email, memberType }` |
| MEMBER_EDITED | `{ description: "Updated member [name]'s profile", memberId, name }` |
| MEMBER_DEACTIVATED | `{ description: "Deactivated member [name]'s account", memberId, name }` |
