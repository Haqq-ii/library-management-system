## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | Information Disclosure | getAuditLog | mitigate | requireRole("LIBRARIAN") enforced as first call — members cannot access audit log |
| T-03-05-02 | Information Disclosure | /audit page | mitigate | Server Component auth guard: role !== "LIBRARIAN" redirects before data fetch |
| T-03-05-03 | Tampering | getAuditLog filter params | mitigate | Date strings parsed with `new Date()` — invalid dates produce NaN and the where clause produces no results (safe failure); action array values cast to AuditAction enum — Prisma rejects invalid enum values |
| T-03-05-04 | Repudiation | AuditLog entries | mitigate | actorId is session.user.id from requireRole() — cannot be client-supplied; createdAt is @default(now()) — cannot be client-set |
| T-03-05-05 | Tampering | AuditLog writes in retrofitted actions | mitigate | All audit writes inside prisma.$transaction with the mutation — no orphaned audit entries if mutation fails |
| T-03-05-06 | Tampering | details JSON field — XSS via description | mitigate | AuditTable renders details.description as text content (React default escaping), not dangerouslySetInnerHTML — no XSS risk |
| T-03-05-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan |
