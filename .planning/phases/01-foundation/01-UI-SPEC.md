## Spacing Scale

Declared values (all multiples of 4px — maps to Tailwind v4 default scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, table cell icon spacing |
| sm | 8px | Compact button padding, form field internal gap, badge gap |
| md | 16px | Default content padding, card body padding, form row gap |
| lg | 24px | Section padding inside slide-over panel, card padding |
| xl | 32px | Page content horizontal padding, sidebar section gap |
| 2xl | 48px | Login card vertical padding, page-level section breaks |
| 3xl | 64px | Reserved for page-level centering margins on wide viewports |

Exceptions:
- Touch targets (buttons, row actions, sidebar links): minimum 44px height per WCAG 2.5.5. Achieved via `min-h-[44px]` on interactive elements — not a new spacing token.
- Sidebar width: 240px collapsed-to-icon = 64px. These are layout dimensions, not spacing tokens.

## Typography

All sizes in px mapped to Tailwind v4 utility classes (`text-sm` = 14px, `text-base` = 16px, `text-xl` = 20px, `text-2xl` = 24px, `text-3xl` = 30px). Font: Inter (system stack).

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 14px | 400 (regular) | 1.4 | `text-sm font-normal leading-snug` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |

**Constraints:**
- Exactly 4 sizes declared (14, 14, 20, 24px). Body and Label share 14px and share weight — differentiation is by size context and layout position only.
- Exactly 2 weights: 400 (regular) for Body and Label; 600 (semibold) for Heading and Display.
- No custom font loading in Phase 1 — Inter via system font stack. Source: CLAUDE.md (Tailwind v4), CONTEXT.md (discretion area).

## Color

Tailwind v4 uses OKLCH tokens via `@theme` directive in `globals.css`. shadcn/ui default slate palette maps as follows:

| Role | Tailwind v4 Token | Hex Approximation | Usage |
|------|-------------------|-------------------|-------|
| Dominant (60%) | `--background` / `slate-50` | `#f8fafc` | Page background, main content area |
| Secondary (30%) | `--card` / `slate-100` | `#f1f5f9` | Sidebar background, card surfaces, table header |
| Accent (10%) | `--primary` / `slate-900` | `#0f172a` | Reserved list below |
| Destructive | `--destructive` / `red-600` | `#dc2626` | Soft-delete (deactivate) actions only |

**Accent reserved for (explicit list — nowhere else):**
1. Primary action buttons: "Add Book", "Save", "Auto-fill" (ISBN), "Add Member"
2. Active sidebar navigation item indicator
3. Login form submit button
4. "Show inactive" toggle when in active state (pressed/on)

**Semantic badge colors (copy status — locked in CONTEXT.md D-specifics):**
| Badge | Color | Tailwind Utility |
|-------|-------|-----------------|
| AVAILABLE | Green | `bg-green-100 text-green-800` |
| CHECKED_OUT | Yellow/Amber | `bg-amber-100 text-amber-800` |
| RESERVED | Blue | `bg-blue-100 text-blue-800` |
| LOST | Red | `bg-red-100 text-red-800` |
| WITHDRAWN | Red (muted) | `bg-red-50 text-red-700` |

These badge colors are semantic indicators, not accent — they live outside the 10% accent budget.

**Destructive color is used for:**
- "Deactivate member" action button/confirmation
- "Delete book" (soft-delete) action in row menu
- Never for navigation, status badges, or informational elements.

Source: CONTEXT.md (Claude's Discretion for color scheme, D-specifics for badge colors).

## Interaction Contracts

### Sidebar Navigation (D-05, D-06, D-07)
- Desktop (≥1024px): sidebar is 240px wide, always visible, pinned left.
- Mobile (<1024px): sidebar collapses to hidden; hamburger icon in top bar opens it as an overlay sheet.
- Active route: accent color left border indicator + `bg-slate-100` background on nav item.
- Inactive route: no background, `text-slate-600` label, `text-slate-500` icon.
- Sidebar footer: logged-in user avatar + name + role chip + sign-out link.

### Login Page (D-08)
- Card is centered horizontally and vertically on a `bg-slate-50` full-screen background.
- Card width: 400px max on desktop, full-width with 16px side padding on mobile.
- Fields: Email, Password. Submit button full-width.
- On submit: button shows loading spinner (disabled state) while Server Action runs.
- On error: inline form error below the password field (shadcn Form error message pattern).
- No "forgot password" in Phase 1 (deferred).

### Slide-Over Panel (D-12, D-13) — Add/Edit Book and Add/Edit Member
- Opens from the right edge. Width: 480px on desktop, full-width on mobile.
- Background page remains visible but is dimmed with `bg-black/40` overlay.
- Panel header: title ("Add Book" / "Edit Book" / "Add Member" / "Edit Member") + X close button.
- Footer: "Save" primary button + "Cancel" text button (closes panel without saving).
- On save: panel closes, table row updates or new row appears, sonner toast confirms success.
- On API error: toast shows error message; panel stays open so user can correct.
- ISBN "Auto-fill" button (book form only): sits inline to the right of the ISBN input. Shows spinner while fetching. On success: fills Title, Author, Publisher, Year fields. If ISBN not found: inline error text below the input ("No book found for this ISBN").

### Catalog Table (D-09) — Librarian Book List
- Columns: Title, Author, ISBN, Copies (total / available count), Genre, Actions.
- Sortable columns: Title, Author, ISBN. Click column header to sort ascending → descending → unsorted.
- Row actions: "Edit" (opens slide-over) + "Deactivate" (opens confirmation dialog). Actions in a `DropdownMenu` triggered by a `…` icon button.
- Deactivate confirmation dialog: title "Deactivate book?", body "This book will be hidden from the catalog. Existing loan records are preserved.", Cancel + "Deactivate" (destructive) buttons.
- "Show inactive" toggle above the table: when on, soft-deleted records appear with `opacity-60` row styling and a "Inactive" badge in the title cell.
- Pagination: 20 rows per page, offset-based. Prev/Next buttons + page indicator ("Page 1 of 3").
- Empty state (no books): centered icon (BookOpen from lucide) + heading "No books yet" + body "Add your first book to start building the catalog." + "Add Book" button.

### Copies Sub-Table (D-10) — Book Detail Page
- Columns: Copy ID (numeric), Status badge, Added date, Actions.
- Actions per copy: "Mark Lost", "Mark Withdrawn" — both open a small confirmation dialog.
- "Add Copy" button above the table adds a new copy record with status AVAILABLE.
- Status badge uses `BookStatusBadge` component with semantic colors (see Color section).

### Member Catalog Search (D-11) — Member Search View
- Prominent search input at top of page (placeholder: "Search by title, author, or ISBN…").
- Results appear as a responsive card grid: 1 column mobile, 2 columns tablet, 3 columns desktop.
- Each card: book cover placeholder icon, title (truncated at 2 lines), author, availability badge.
- No results state: centered icon (Search from lucide) + "No books found" + "Try a different search term."
- Search is client-side filtered with debounce (300ms) on the loaded seed dataset; no pagination required for Phase 1.

### Soft-Delete Visibility Toggle
- Present on `/books` and `/members` pages.
- Uses shadcn `Switch` component with label "Show inactive".
- Default off (only active records shown). When toggled on: inactive rows/cards appear with visual muting (`opacity-60`, italic title text).

### Loading States
- Table loading: full table rows replaced with `Skeleton` rows (3–5 rows, matching column count).
- Card grid loading: card-shaped `Skeleton` placeholders.
- Form submitting: primary button enters disabled + spinner state via `useFormStatus` or manual `isPending` flag.

## Accessibility Contracts

- All interactive elements: minimum 44×44px touch target (WCAG 2.5.5).
- Form fields: `<label>` elements associated via `htmlFor` / shadcn Form pattern — never placeholder-only labels.
- Color is never the sole differentiator: status badges include text label alongside color.
- Sidebar navigation: `aria-current="page"` on active link.
- Mobile sidebar trigger: the hamburger icon button must carry `aria-label="Open navigation"` — it is icon-only and has no visible text.
- Confirmation dialogs: focus trapped inside dialog; Escape key closes. Managed by Radix Dialog primitive.
- Destructive buttons: labeled with action verb, not just "OK" or "Yes".
- Skip-to-main-content link in `<head>` area (hidden until focused) — single implementation, applies to all pages.
