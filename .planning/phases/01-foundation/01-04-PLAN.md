<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → login form | Untrusted credentials submitted |
| middleware → app route | UX gate only, bypassable (CVE-2025-29927) |
| session cookie → shell | Session presence gates rendering |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | `middleware.ts` | mitigate | Middleware redirect is explicitly UX-only (commented); the real boundary is `requireRole()` in Server Actions + `getSession()` in `(app)/layout.tsx`. Never trust middleware for auth (CVE-2025-29927) |
| T-04-02 | Spoofing | open redirect post-login | mitigate | Post-login routing is a fixed allowlist (`/dashboard` or `/catalog`) derived from role, not a user-supplied `callbackUrl`; Better Auth `trustedOrigins` (Plan 02) backs this (T-05) |
| T-04-03 | Information Disclosure | login error message | mitigate | Generic "Incorrect email or password" copy (UI-SPEC) — does not reveal whether the email exists (no user enumeration) |
| T-04-04 | Information Disclosure | dashboard stat queries | mitigate | Stat counts run in a session-gated Server Component; queries filter `deletedAt: null`; no per-user PII exposed |
</threat_model>
