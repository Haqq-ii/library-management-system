<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → catalog Server Action | Untrusted form input + caller role |
| server → Open Library API | Untrusted external API response |
| action → Prisma | Mass-assignment surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | catalog actions | mitigate | `requireRole("LIBRARIAN")` first line of every action (T-01); unit test asserts MEMBER rejection. Not middleware (CVE-2025-29927) |
| T-05-02 | Tampering | mass assignment | mitigate | Zod `BookSchema.safeParse` with explicit fields; never spread FormData into `prisma.book.create` (T-02) |
| T-05-03 | Tampering | Open Library response injection | mitigate | API JSON parsed through a Zod schema before any DB insert (T-06); only whitelisted fields used |
| T-05-04 | Tampering | hard delete | mitigate | Soft delete only (`deletedAt: new Date()`); `prisma.book.delete(` banned (INFRA-04); acceptance criterion greps for absence |
| T-05-05 | Information Disclosure | action errors | mitigate | Typed `ActionResult` returned; `console.error` server-side only; no stack traces or raw errors to client (V7) |
</threat_model>
