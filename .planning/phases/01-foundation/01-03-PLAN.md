<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → container | Volume mounts expose host files to container |
| container → Postgres | DDL + seed writes cross this boundary at startup |
| git → image | Env files must not leak secrets into the committed tree or image |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | env files in image/git | mitigate | Real `.env.*` gitignored (Plan 01); only `.env.example` with empty values committed; staging/prod files carry placeholders, never real secrets |
| T-03-02 | Denial of Service | seed on every restart | mitigate | D-03 conditional seed (psql User-count guard) prevents duplicate seeding and data clobbering on restart |
| T-03-03 | Tampering | migration order | mitigate | Entrypoint runs `prisma generate` before `migrate deploy` (Pitfall 2); acceptance criterion enforces line ordering |
| T-03-04 | Elevation of Privilege | default Postgres credentials | accept | Dev-only `postgres/postgres` on an isolated Docker network not exposed beyond localhost; production uses managed Postgres creds via real env config |
</threat_model>
