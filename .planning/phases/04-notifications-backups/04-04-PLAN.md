<interface_note>
CRITICAL schema fact: `NotificationLog` (schema.prisma lines ~227-235) has fields `id, memberId (String), type (String), sentAt (DateTime), channel (String), success (Boolean), metadata (Json?)`. It has NO `member` relation and NO `createdAt` field. Therefore:
- Order and display by `sentAt`, NOT `createdAt` (the AuditTable analog uses `createdAt` — do not copy that field name blindly).
- Member name cannot be fetched via a Prisma `include`. Resolve names by collecting the page's `memberId`s and running a separate `prisma.member.findMany({ where: { id: { in: ids } }, include: { user: true } })`, then mapping memberId -> user.name/email for display. Rows whose member lookup misses fall back to showing the raw memberId.
</interface_note>
